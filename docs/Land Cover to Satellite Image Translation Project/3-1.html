Data Acquisition

The data acquisition process involves preparing the datasets required for training the Pix2Pix model. This section outlines the steps to define the study area, generate random sampling points within the area, and fetch paired satellite and land cover images using geospatial tools like Google Earth Engine. These steps ensure a diverse and representative dataset, which is crucial for effective model training and testing.

Step 1

In this study, I aim to train and generate images based on random locations within specific areas with desired landscapes. To achieve this, I prepared a shapefile containing the research area in polygon format in ArcGIS and selected random locations within this polygon. The dataset includes 90% of images from the barrier island area in Tamaulipas, Mexico, and 10% from Philadelphia, to ensure diversity in land cover types and improve model performance.

The barrier island area was chosen for its rich diversity in land cover types, such as wetlands, forests, and open land, which provide a wide range of training data for the model. However, initial tests using only this data revealed that the model’s performance was not ideal in developed urban areas. To address this limitation, I included 10% of the training data from Philadelphia, as it’s dominated by developed land, such as buildings and roads. This additional urban data complements the natural land cover diversity from Mexico, enabling the model to generalize more effectively across both natural and developed environments. Step 2

Then, we generate 1000 random points within the area. These points are then stored to a json file at coords. The num_points variable can be adjusted to generate any number of random points. The num_points_per_polygon variable can be calculated differently to adjust the number of points generated in each shapefile.

def generate_random_points_all(polygon1, polygon2, total_num_points):
    def generate_points(polygon, num_points):
        minx, miny, maxx, maxy = polygon.bounds
        points = []
        attempts = 0
        max_attempts = num_points * 10  
        while len(points) < num_points and attempts < max_attempts:
            random_point = Point(random.uniform(minx, maxx), random.uniform(miny, maxy))
            if polygon.contains(random_point):
                points.append({'latitude': random_point.y, 'longitude': random_point.x})
            attempts += 1
        return points
    
    num_points_per_polygon = total_num_points * 0.9

    points1 = generate_points(polygon1, num_points_per_polygon)
    points2 = generate_points(polygon2, total_num_points - num_points_per_polygon)  

    return points1 + points2

Step 3

In the final step, we fetch one satellite (true-color) image and one land cover image for each randomly selected location. This process involves the following detailed steps:

- Directories Setup: The directories for storing true-color and land cover images are created if they do not already exist. The images are saved in ./24-1204_training_images_2/RC for true-color and ./24-1204_training_images_2/LC for land cover images, ensuring an organized structure.

- Image Region Definition: For each point, its latitude and longitude coordinates are used to define a geographical region of interest. A buffer around the point is created based on the image size (1024x1024 pixels) and pixel resolution (10 meters), resulting in a bounding box that ensures spatial consistency for all images.

- Fetching True-Color Satellite Images: The true-color images are sourced from the Landsat 8 Collection (LANDSAT/LC08/C02/T1_TOA). The following filters are applied: Location: Images covering the specific point. Date Range: Images from the year 2022. Cloud Cover: Sorted to select the image with the least cloud cover. If an image is found, its RGB bands (B4, B3, B2) are selected and visualized using predefined minimum and maximum reflectance values. A thumbnail URL is generated to be downloaded in PNG format. The downloaded image is saved with a sequential filename (e.g., image_0.png).

- Fetching Land Cover Images: The land cover data is retrieved from the ESA WorldCover dataset (ESA/WorldCover/v100/2020). The dataset is clipped to the defined region, and a custom color palette is applied to differentiate various land cover types (e.g., forests in dark green, built-up areas in red). A thumbnail of the land cover image is generated and downloaded in PNG format, then saved with a corresponding filename in the designated directory.

- Error Handling: The process includes error handling to ensure robustness. If an image cannot be fetched or downloaded (e.g., due to unavailability or network issues), the script logs the failure and moves to the next point without stopping.

- Iteration Over Points: The process_point function is executed for each coordinate in the coords_points list, iterating over all selected locations. This ensures that both true-color and land cover images are collected for every point in the dataset.
